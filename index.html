    • 字符函数

      • ASCII()

        返回一个字符串的第一个字符的ASCII码，他的逆函数是CHR()

      • CHR()

      ASCII()的逆函数

* INITCAP()

      返回字符串的每个单词的第一个字母大写而单词中的其他字母小写的string。

* LTRIM()

      反回删除从左边算起出现在string2中的字符的string1。String2被缺省设置为单个的空格。

* RTRIM()

      返回删除从右边算起出现在string1中出现的字符string2. string2被缺省设置为单个的空格.

* TRIM()

      删除前后的空格

* UPPER()

      返回大写的string.不是字母的字符不变.CHAR返回CHAR，VARCHAR2返回VARCHAR2。

* SUBSTR(string,a[,b])

      返回从a开始b个字符长的string的一个子字符串.
      a为负数表示从后往前数，b为空表示取到末尾，b为0或负数返回null

* REPLACE(string,search_string[,replace_string])

      将string中的search_string用replace_string删除,如果replace_string为空，则删除。

* LPAD(string1,n,string2)

      用string2将string1从左边进行填充，填充到长度为n

* LENGTH（string）

      返回string的字符单位的长度.CHAR数值是填充空格类型的,如果string由数据类型CHAR,它的结尾的空格都被计算到字符串长度中间.如果string是NULL,返回结果是NULL,而不是0.

  • 数字型

    • FLOOR(n)
      返回小于等于n的最大整数

* ROUND(x[,y])

      计算保留到小数点右边y位的x值. y缺省设置为0,这会将x保留为最接近的整数.如果y小于0,保留到小数点左边相应的位. Y必须是整数.进行四舍五入。

* CEIL(n)

      向上取整

* TRUNC(x[,y])

      计算截尾到y位小数的x值. y缺省为0,结果变为一个整数值.如果y是一个负数,那么就截尾到小数点左边对应的位上. 只是该函数不对指定小数前或后的部分做相应舍入选择处理，而统统截去。

  • 日期函数

    默认的日期格式可以在每个会话中更换：
    ALTER SESSION SET NLS_DATE_FORMAT = ‘DD_MM_YYYY HH24:MI:SS’

    * TO_DATE(String[,format[,nlsparams]])

            把CHAR或者VARCHAR2类型的String转换为一个DATE类型的变量. format是一个日期格式字符串.当不指定format的时候,使用该会话的缺省日期格式.
            例子：to_date('2020-1-1 13:20:00','YYYY-MM-DD HH24:MI:SS')
    * TO_CHAR(d [,format[,nlsparams]])

            将日期d转换为一个VARCHAR2类型的字符串.如果指定了format,那么就使用位置:它控制结果的方式.格式字符串是由格式元素构成的.第一个元素返回日期数值一个部份,例如日子.如果没有给定format,使用的就是该会话的缺省日期格式.如果指定了nlsparams,它就控制着返回字符串的月份和日分量信息所使用的语言. 
            nlsparams的格式是: “NLS_DATE_LANGUAGE”
            例子: to_char(sysdate,'YYYY-MM-DD HH24:MI:SS')
    * SYSDATE

            返回当前的时间

    *  TRUNC(d,format)

            计算截尾到由format指定单位的日期d.可以使用位置:格式和效果.缺省参数同ROUNG.
    * ADD_MONTHS(d,x)

            返回日期d加上x个月后的月份。x可以是任意整数。如果结果日期中的月份所包含的天数比d日期中的“日”分量要少。（即相加后的结果日期中的日分量信息已经超过该月的最后一天，例如，8月31日加上一个月之后得到9月31日，而9月只能有30天）返回结果月份的最后一天。
            返回NUMBER类型
    * LAST_DAY(d)

            计算包含日期的d的月份最后一天的日期.这个函数可以用来计算当月中剩余天数.
    * MONTHS_BETWEEN(date 1,date2)

            计算date 1和date2之间月数.如果date 1,date2这两个日期中日分量信息是相同的,或者这两个日期都分别是所在月的最后一天,那么返回的结果是一个整数,否则包括一个小数,小数为富余天数除以31.

mongodb

post @ 2021-01-03

  • 重要概念

    • mongodb结构

      关系数据库 mongodb
      数据库 数据库
      表 集合
      行 文档

    • 集合

      • 集合是无模式的，集合中的文档的格式可以各不相同。表需要提前设计表的结构；而集合不需要设计结构，任何格式的文档都可以存储进集合。

    • 文档

      • 文档的数据结构和 JSON 基本一样。所有存储在集合中的数据都是 BSON 格式。
      • BSON 是一种类似 JSON 的二进制形式的存储格式，是 Binary JSON 的简称。

    • 文档例子

      {name: ”Tom”, 
age: 18, 
hobbies:  [“football”,”swimming”]}

  • 基础操作

    • 查看所有的数据库
      shwo dbs
    • 创建数据库(如果数据库不存在，则创建数据库，如果存在，则切换到该数据库。新创建的数据库不会出现在列表中，需要向其中插入数据才能在该命令的结果中看到。)
      use db_name
    • 查看当前使用的数据库
      db
    • 删除当前数据库
      db.dropDatabase()
    • 集合
      • 创建集合
        db.createCollection("books")
      • 查看数据中的集合
        show tables
      • 删除集合
        db.books.drop()
      • 向不存在的集合插入文档时会自动创建集合
        db.mycol2.insert({"name" : "dhm"})
        自动创建集合mycol2
    • 文档
      • 插入文档
        db.COLLECTION_NAME.insert(document)

  • CURD操作

    • 插入

      • db.collection.insertOne() 可插入一行，参数为文档
        db.products.insertOne( { item: "card", qty: 15 } );
      • db.collection.insertMany() 可插入多行，参数为文档数组
        db.products.insertMany( [ { item: "card", qty: 15 }, { item: "envelope", qty: 20 } ] );
      • db.collection.insert() 可插入一行或多行，参数为文档或文档数组
        可以使用insert插入一个或多个文档，取决于传入的参数是文档还是文档数组。

    • 查询

      • db.collection.find(query, projection)

      • 查询所有文档
        db.inventory.find( {} )

      • 对文档进行格式化
        db.inventory.find( {} ).pretty()

      • 匹配查询
        db.inventory.find( { status: "D",qty:75 } )

      • 嵌套文档查询，顺序必须相同

      • 比较操作符
        $gt 大于
        $lt 小于
        $gte 大于等于
        $lte 小于等于
        $ne 不等于
        $eq 等于
        $in 符合其中任意一个，即可检索出
        $nin 不符合才会检索出

      • 例子
        db.inventory.find( { qty:{$gt:50}} )
        qty大于50
        db.products.find({status:{$ne:”D”}})
        status不等于D

      • 逻辑操作符

        • 格式：
          {操作符:[{条件1}，{条件2}....]}
        • 隐式and
          db.inventory.find( { status: "A", qty: { $lt: 30 } } )
        • 显式and
          db.inventory.find( { $and: [ { price: { $ne: 1.99 } }, { price: { $exists: true } } ] } )
        • OR
          db.inventory.find({ $or: [ { status: "A" }, { qty: { $lt: 30 } }]} )
        • not
          db.inventory.find( { price: { $not: { $gt: 1.99 } } } )
        • exists (查询文档中包含查询字段的值,包括null)
          db.inventory.find( { qty: { $exists: true, $nin: [ 5, 15 ] } } )
        • type（查询字段值得类型为某个或某些BSON type的文档。）
          db.inventory.find( { status: { $type:”string” } } )
          db.inventory.find( { status: { $type:”null” } } )

      • 精确查询(包括数组中的顺序)
        db.inventory.find( { tags: ["red", "blank"] } )

      • $all 忽略元素顺序
        db.inventory.find( { tags: { $all: ["red", "blank"] } } )
        tag数组中包含red,blank,且不限顺序

      • $size(使用数组长度查询)
        db.inventory.find( { "tags": { $size: 3 } } )

      • $elementMatch (满足一个条件即可)
        db.inventory.find( { dim_cm: { $elemMatch: { $gt: 22, $lt: 30 } }})

      • 数组索引
        db.inventory.find( { "dim_cm.1": { $gt: 25 } } )
        dim_cm中第一个值大于25

      • 游标

        • 创建游标
          var mycursor=db.inventory.find()

        • 访问游标
          mycursor[4]

        • 遍历完游标中的所有文档后，或10分钟后，游标会自动关闭

        • var mycursor=db.inventory.find().noCursorTimeOut()

          改变游标默认行为，不自动关闭

        • 手动关闭游标
          mycursor.close()

        • count()
          接受一个参数，默认为false表示不考虑skip和limit函数的影响；为true时表示考虑这两个函数的影响。
          db.orders.find().count()
          返回文档数量

        • limit() 指定游标返回的最大文档数 （limit(0)与没有使用limit效果一样）
          db.orders.find().limit(5).count(true)

      • 投射
        指定查询结果中出现的字段

        • 0：不出现，1：出现，_id列默认出现
          db.inventory.find( { status: "A" }, { item: 1, status: 1, "size.uom": 1 })

    • 修改

      • db.collection.updateOne() 更新最多一篇文档 第二个参数：更新操作符的文档
      • db.collection.updateMany() 更新多篇文档 第二个参数：更新操作符的文档
      • db.collection.replaceOne() 用新文档替换原来的文档 第二个参数：新的文档
      • db.collection.update() 更新或替换原来的文档 第二个参数：更新操作符的文档或新文档
      • 文档主键_id是不能更改的
      • 操作符
      $set 更新或新增字段
      $unset 删除字段
      $rename 重命名字段
      $inc 加减字段值
      $mul 相乘字段值
      $min
      $max

      • $set

        • 普通修改

          db.products.update(
{ _id: 100 },   //查询_id的100的文档
{ $set: { details: "zzz" } } )//修改details为zzz

        • 嵌套文档修改

          db.products.update(
{ _id: 100 },   
{ $set: { "details.make": "zzz" } } )//{details:{make:'zzz'}}

        • 修改数组

          db.products.update(
{ _id: 100 },
{ $set: {
"tags.1": "rain gear",
"ratings.0.rating": 2      }   })// {rating:[{rating:2}]} 数组中第一个元素的rating改为2

      • $unset

        db.products.update(
{ _id:100 },
{ $unset: { quantity: "", "details.model": "" } })
删除数组中的元素时，数组的长度不变，只是把相应位置的值改为null：

      • $rename

        重命名文档中的字段
        db.students.updateMany( {}, { $rename: { "nmae": "name" } } )
        重命名内嵌文档中的字段
        db.students.update( { _id: 1 }, { $rename: { "name.first": "name.fname" } } )
        改变字段位置
        db.students.update( { _id: 1 }, { $rename: { "name.last": "last ","mobile": "name mobile"} } )

      • $inc
        db.students.update( { _id: 1 }, { $inc: { "score": -10} } )

      • $mul
        db.students.update( { _id: 1 }, { $mul: { "score": 0.5} } )

    • 删除

      • db.collection.deleteOne() //删除满足筛选条件的一篇文档
      • db.collection.deleteMany() //删除满足筛选条件的所有文档
      • db.collection.remove() //删除满足筛选条件的一篇或所有文档，取决于justOne的取值
      • 删除集合中的所有文档
        db.students.deleteMany({})
      • 删除整个集合
        db.students.drop()

crontab

post @ 2021-01-03

介绍

执行定时任务，新创建的 cron 任务，不会马上执行，至少要过 2 分钟后才可以，当然你可以重启 cron 来马上执行。

而 linux 任务调度的工作主要分为以下两类：

  • 1、系统执行的工作：系统周期性所要执行的工作，如备份系统数据、清理缓存
  • 2、个人执行的工作：某个用户定期要做的工作，例如每隔10分钟检查邮件服务器是否有新信，这些工作可由每个用户自行设置

语法

  • crontab [-u user] file

> -u 指定设置谁的定时任务，需要具有相应的权限，不指定默认设置自己的。

  • 参数说明

    • -e : 执行文字编辑器来设定时程表，内定的文字编辑器是 VI，如果你想用别的文字编辑器，则请先设定 VISUAL 环境变数来指定使用那个文字编辑器(比如说 setenv VISUAL joe)
    • -r : 删除目前的时程表
    • -l : 列出目前的时程表

  • 时间格式

    f1:分钟 ,* 表示每分钟都要执行,a-b表示a-b这段时间执行,*/n表示每n分钟执行一次

    f2:小时

    f3:每月中的第几日

    f4:月份

    f5:每星期中的第几天

    program:执行的程序
